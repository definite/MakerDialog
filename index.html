MakerDialog by definite

Utility that conveniently handles configuration properties

MakerDialog is an utility that conveniently handles configuration properties by producing corresponding UI widgets and load/save functions. It generates corresponding UI widgets according to the specification of configuration properties, such as data type, default value, valid values and limitation. Developers no longer need to worry about the event handling details of various toolkit.

Dependencies

  • Main package:
    • glib >=2.12
    • CMake >=2.4 (not required for runtime)
    • doxygen (not required for runtime)
    • pkgconfig (not required for runtime)
  • Gtk binding:
    • gtk2 >=2.10
    • gob2 (not required for runtime)
  • GConf2 binding:
    • GConf2

Installation instruction

Building stage

  1. Download and extract source tarball from the download page.
  2. Enter the extracted directory.
  3. Run: 
    cmake . -DCMAKE_INSTALL_PREFIX='/usr'
    make
  4. (Optional) If API documents are needed, then: 
    make doxygen
  5. (Optional) Build the example programs: 
    make examples
  6. (Optional) Test the build: 
    make test

Installation/packing stage

Install libraries, header files and pkgconfig files of MakerDialog and all the bindings. 

make install

Make RPM packages for MakerDialog: 

make rpm

Run rpmlint on built RPMs: 

make rpmlint

See more make targets by: 

make help

Trouble shooting

If gtk2 binding fail to build, it is usually because of the strcpy bug in gob2 (see here for details). Thus, MakerDialog is packed with generated source codes: 

src/gtk/maker-dialog-gtk.c
src/gtk/maker-dialog-gtk.h
src/gtk/maker-dialog-gtk-private.h

Should you use make clean to clean but cannot build them back, please either:

  • Extract these files from the tarball; or
  • Patch gob2 accordingly.
